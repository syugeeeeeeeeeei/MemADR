# 実装計画

## 方針

ヘルプ文言を場当たり的に `switch` 文へ埋め込まず、コマンド定義を1か所に集約し、その定義から総合ヘルプと個別ヘルプを描画する。これにより、コマンドの追加時に分岐とヘルプ文言の二重更新を減らす。

## 実装順

1. 引数なし実行時にヘルプが出ることをテストで固定する
2. `memadr help new` の詳細ヘルプをテストで固定する
3. コマンド定義とヘルプレンダラを追加する
4. `Run` の分岐を定義ベースに置き換える
5. ドキュメントと memory レコードを更新する

## 設計

- `internal/app/help.go`: コマンド定義、総合ヘルプ、個別ヘルプの描画
- `internal/app/run.go`: 引数解釈とコマンド実行
- `internal/mem/kind.go`: レコード種別説明を保持し、ヘルプに流用する
