# ADR-006: GitHub Release生成をタグpush起点のActionsへ委譲する

Status: ACCEPTED
Context: ローカル `just release` だけでバイナリ作成と GitHub Release 公開まで完結させると、配布物の再現性とマルチプラットフォーム対応が環境依存になりやすい
Decision: ローカル側は版検証、テスト、タグ作成、push までを担当し、GitHub Actions はタグ push を契機にマルチプラットフォームバイナリ作成と GitHub Release 作成を担当する
Rejected: ローカル環境だけで全OS向け配布物を毎回作って手動アップロードする案
Consequence: 開発者の入口は `just release vX.Y.Z` に揃えつつ、配布物はCI上で再現可能に生成できる
