# ADR-007: release再実行時は既存タグをHEADへ再配置する

Status: ACCEPTED
Context: プレリリースや修正版リリースでは同じ版名でタグを現在コミットへ更新したいことがあり、既存タグで即失敗すると運用が止まりやすい
Decision: `just release vX.Y.Z` 再実行時は、既存タグを削除エラーにせず現在の `HEAD` へ付け直し、origin へ force-push して release workflow を再実行する
Rejected: 既存タグがあるたびに人手で削除してから再実行する案
Consequence: 同じ版名の再発行は容易になる一方で、タグ履歴を書き換えるため release コマンドの副作用は明確に維持する
