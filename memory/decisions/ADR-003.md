# ADR-003: レコード処理を共有パーサー基盤に集約する

Status: ACCEPTED
Context: `check` `list` `search` `index` `related` `close` は同じMarkdown正本を読み、別々にパースすると仕様差と保守コストが増える
Decision: CLI層の下に共有の `record` パッケージを置き、読込、検査、検索、関連候補判定、集約生成、部分更新をそこで扱う
Rejected: コマンドごとに個別のMarkdown解釈を持つ案
Consequence: 依存方向を `app -> record -> mem` に揃えつつ、初期版コマンドを小さい関数群へ分割できる
