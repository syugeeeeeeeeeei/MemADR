# ADR-005: バージョン正本をGit tagとリリース導線へ集約する

Status: ACCEPTED
Context: アプリ版を管理したい一方で、コード定数や VERSION ファイルへ値を書くと、Git tag や配布物との二重管理になりやすい
Decision: 正式版の正本は Git tag とし、`version` コマンドはビルド時注入値を表示する。タグ付与は `release` 導線へ寄せ、人手で個別に `git tag` を打たなくても済むようにする
Rejected: コード定数に版を持つ案、VERSION ファイルを別管理する案、タグ運用を人手に委ねる案
Consequence: 通常ビルドは `dev+commit`、リリースビルドはタグ版を表示し、版管理とリリース操作の入口を揃えられる
