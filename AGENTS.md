# ルール
- ADRを使用して効率的なドキュメントチケット管理を行うツール「MemADR」開発にあたって、この開発自体に**ADR**の仕組みを使用する
- 開発は**テスト駆動開発**で行う
- 関数名・変数名は**わかりやすさを保ったまま短め**にし、コード量を減らすようにする
- モジュールの参照の向きを重視し、上位から下位へという依存方向がわかりやすい開発アーキテクチャにする
- 一つのファイルに責任を押し付けすぎず、ファイルの分割を行い、単位ファイルあたりの行数を減らす

## MemADR運用ポリシー

このリポジトリでは、MemADRを開発判断メモリとして使用する。
作業するエージェントは、必ず `MEMADR_WORKFLOW.md` を読み、その内容に従うこと。
特に、次を守ること。

* `memory/` に作業ログを書かない
* 現在も価値がある情報と、過去情報を区別する
* 古い判断、削除済み機能、無効化済み情報を現在有効な前提として扱わない
* MemADRレコードを追加または更新した場合は、完了前に `memadr check` と `memadr index` を実行する
